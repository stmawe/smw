**IMPORTANT:** This file contains guidelines for how Copilot and developers should organize files in this project.

## Quick Rules

### Test/Debug Files в†’ `/debug/` folder
ALL test and debug files MUST be created in the `/debug/` folder, NOT in the project root.

**Examples of files that belong in `/debug/`:**
- `test_*.py` - Any test script (e.g., `test_admin_login.py`)
- `verify_*.py` - Verification scripts (e.g., `verify_deployment.py`)
- `check_*.py` - Check scripts (e.g., `check_database.py`)
- `debug_*.py` - Debug utilities (e.g., `debug_auth.py`)
- `diagnostic_*.py` - Diagnostic tools (e.g., `diagnostic_celery.py`)
- `temp_*.py` - Temporary files (e.g., `temp_migration.py`)
- `*.sh` test scripts - Shell test files (e.g., `test_server.sh`)
- `scratch_*.py` - Scratch files for experimentation

## Rules for When Copilot Creates Files

### When Creating a Test/Debug Script

**вќЊ WRONG:**
```python
# test_feature.py (in root - NOT GOOD)
def test_something():
    pass
```

**вњ… CORRECT:**
```python
# debug/test_feature.py (in debug folder - CORRECT)
def test_something():
    pass
```

**Copilot should:**
1. Always create test files in `debug/` folder
2. Use full path: `debug/test_feature.py`
3. Never create test files in project root
4. Remind user with message: "Created debug/test_feature.py (test files go in debug/ folder)"

### When Creating Documentation

**вќЊ WRONG:**
```markdown
# SETUP_GUIDE.md (in root - NOT GOOD)
# How to set up the project
```

**вњ… CORRECT:**
```markdown
# md/SETUP_GUIDE.md (in md folder - CORRECT)
# How to set up the project
```

**вњ… EXCEPTION - README:**
```markdown
# README.md (in root - CORRECT for README only)
# Main project documentation
```

**Copilot should:**
1. Create documentation in `md/` folder (unless it's README.md)
2. Use full path: `md/SETUP_GUIDE.md`
3. Only create README.md in root
4. Remind user: "Created md/SETUP_GUIDE.md (docs go in md/ folder, except README.md)"

## Gitignore Configuration

The `/debug/` and `/md/` folders are in `.gitignore`:

```gitignore
# Debug scripts and temporary setup files
debug/

# Documentation folder (archived docs not needed in repo)
md/
```

This means:
- Files in these folders are NOT committed to GitHub
- They exist only on local machines
- Developers can safely add test files without committing them
- `.gitkeep` files are force-added to keep folder structure
